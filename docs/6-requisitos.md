# Requisitos

## Requisitos funcionais

* RF01: O sistema deve permitir o cadastro, consulta, edição e exclusão das campanhas;
* RF02: O sistema deve permitir o cadastro, consulta, edição e exclusão das ações;
* RF03: O sistema deve permitir o cadastro, consulta, edição e exclusão dos voluntários;
* RF04: O sistema deve permitir o cadastro, consulta, edição e exclusão dos colaboradores;
* RF05: O sistema deve possibilitar a criação de métricas de desempenho para as campanhas;
* RF06: O sistema deve possibilitar a criação de métricas de desempenho para as ações;
* RF07: O sistema deve realizar o cálculo automático das métricas de desempenho cadastradas;
* RF08: O sistema deve permitir a seleção de filtros (ex: por período, tipo de campanha, região);
* RF09: O sistema deve restringir o acesso às funcionalidades conforme o perfil do usuário;
* RF10: O sistema deve disponibilizar os dados de maneira aberta pela plataforma de visualização de dados;
* RF11: O sistema deve fornecer um formulário, em uma página de acesso público, para que usuários interessados em se tornar voluntários, registrem os seus dados;
* RF12: O sistema deve apresentar uma página pública de doações com QR Code ou uma chave para doação;


## Requisitos não funcionais

* RNF01: A interface deve ser responsiva e acessível em computadores
* RNF02: O sistema deve implementar autenticação segura (OAuth2, JWT ou similar)
* RNF03: O sistema deve ser compatível com os principais navegadores modernos (Chrome, Firefox, Edge, Safari)
* RNF04: O sistema deve garantir que dados pessoais sejam armazenados de forma criptografada, em conformidade com a LGPD.
* RNF05: O sistema deve ter um tempo de resposta menor que 5ms
* RNF06: O sistema deve garantir disponibilidade mínima de 99%, exceto em períodos programados de manutenção.
